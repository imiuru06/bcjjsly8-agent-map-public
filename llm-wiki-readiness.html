현재 결론

Agent Map은 LLM-Wiki의 기반은 갖췄지만, 아직 “검색 ROI가 검증된 RAG/MCP 시스템”은 아닙니다. 지금은 capture → sanitize → visible artifact → agent loop → synthesis로 가는 MVP를 강화해야 합니다.

평가 차원

Zero-friction capture

strong · 15/15

강점: Kakao, GitHub issue, session-agent, and runtime artifacts already act as capture channels.

갭: Capture is still source-specific and does not yet produce a unified inbox note stream.

다음: Create a normalized capture inbox schema before adding heavier RAG infrastructure.

Public/private safety boundary

strong · 15/15

강점: The project has a strong public-safe artifact pattern: raw messages, room ids, raw URLs, and secrets stay out of public pages.

갭: Private local views and public views are not yet first-class separate surfaces.

다음: Keep the public-safe layer, but add explicit private/local-only retrieval contracts.

Atomic knowledge vault

strong · 15/15

강점: Obsidian-style atomic notes, frontmatter, wikilinks, and lookup modes already exist.

갭: Captured operational outputs are not yet automatically converted into durable atomic notes.

다음: Add capture-to-note promotion rules with provenance and stale-note checks.

Retrieval and graph index

strong · 15/15

강점: Graphify is already treated correctly as a derived index, not the source of truth.

갭: A static retrieval pilot exists before full vector DB/RAG, but real search/reuse events are still needed.

다음: Use the static retrieval pilot to collect cited-answer and reuse events before evaluating BGE-M3 + Qdrant.

Agent execution loop

strong · 15/15

강점: The repo already records loop evidence and separates bounded actions from human-gated work.

갭: Actual replacement-ready life domains are still not achieved.

다음: Convert repeated low-risk loops into measurable replacement candidates before claiming agentization.

Synthesis and feedback

strong · 10/10

강점: Weekly synthesis exists as a principle/runbook and life-loop feedback exists as runtime evidence.

갭: Synthesis is not yet a consistently scheduled, user-facing weekly artifact.

다음: Create a weekly synthesis artifact that links new captures, decisions, and knowledge gaps.

Search and reuse ROI measurement

partial · 7/10

강점: Runtime artifacts make future ROI measurement possible.

갭: Search/citation/reuse events are now logged, but the sample size is still too small for durable ROI conclusions.

다음: Keep logging real search, cited-answer, reused-note, and action-conversion events before investing in full RAG/MCP automation.

Integrated UX visibility

strong · 5/5

강점: Agent Map already exposes public results, infrastructure, knowledge, and Kakao analysis in one deployed surface.

갭: The UX is still more dashboard/index than true knowledge workbench.

다음: Add one workbench-style path: ask → retrieve evidence → show source → create issue/action.

구현 방향

  • Keep Graphify/RAG/MCP as derived capability layers, never source of truth.
  • Prioritize measured reuse: search count, cited answer count, action conversion, and stale-note reduction.
  • Preserve human-gated boundaries while letting agents run bounded evidence-producing loops.
