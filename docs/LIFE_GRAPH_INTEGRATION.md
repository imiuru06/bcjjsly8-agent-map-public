`agent-map-life.html` is generated by the React/Vite Life Graph frontend for explicit access, but `/` is the canonical route.

## Why keep v9 temporarily

The current `public/agent-map-v9.html` is kept as an internal Registry/runtime diagnostics view. It should not be exposed as public product navigation, but it is useful while the life-process graph becomes the service-facing Agent Map.

Use it only for:

- Registry snapshot verification
- runtime sync debugging
- deployment fallback diagnostics

Once the Life Graph includes enough operational status panels, `agent-map-v9.html` can be removed or converted into an authenticated/admin-only tool.

## Data model

The first-pass runtime contract is JSON-only and does not require Neo4j or any external graph DB.

```text
runtime/life-process-graph.json
```

Top-level sections:

- `metaProcess`
- `tracks`
- `lifeDomains`
- `agents`
- `pipelines`
- `relationships`
- `currentness.snapshots`

The UI derives graph nodes and edges from those sections, and now composes public-safe supporting snapshots:

```text
runtime/decision-queue.json
runtime/session-agent-health.json
config/substitute-agent-domains.json
```

## Graph modes

`agent-map-life.html` now renders the Agent Map Service Map with React Flow + ELK and supports the restored graph concept through multiple views:

- Force
- Domain Cluster
- Process Flow
- Pipeline Trace
- Phase Lane

This keeps the old graph interaction concept while using a smaller `bcjjsly8`-native runtime contract.

## Safety constraints

- Do not expose `public/agent-map-v9.html` as public product navigation.
- Do not replace `runtime/agent-map-runtime.json`.
- Do not require external graph DB for the first pass.
- Do not put private life data or secrets into public runtime JSON.
- Treat #18 as the durable coordination issue.

## Current first-pass extensions

- Life Graph currentness references the session-agent health snapshot, decision queue, substitute-agent domain contracts, and throughput policy.
- The top-level Decision Queue panel reads `runtime/decision-queue.json` before falling back to issue nodes embedded in the graph.
- `npm run agent:system:check` validates public-safe snapshot boundaries and prevents raw session messages, secrets, or executed decisions from entering these public artifacts.
